Syntax of CSV file fields

The input file for import is in the same CSV format as the one exported. The best way to understand the CSV format is to create few users and export the data.

  • Operation: Mandatory - 'c' (create), 'C' (create with default password), 'u' (update), 'U' (update with default password), 'e' (send password email only), 'E' (configure random password without sending mail), or 'i' (ignore). Each row when importing could be an instruction to create, update (including deactivate), or notify a user. 'C'/'U' option creates/updates a user with default password = login user name. This does not generate registration mail. The user can login with username='user_id.groupid' and password='user_id.groupid')
  • Key: Mandatory - is of format "user id.group id". Only in the case the user is modified and the user id is to be changed, key is useful. In such a case, key is the old "user id.group id" and user id is the new one. Nittio Admin has possibility to move a user from one group to another. In such a case, the import has to be done from the target group and the key should be of form "userid.sourcegroupid".
  • User Id: Mandatory - Unique id within the group - employee Id will be a good fit; Has to be atleast 1 character long and has only characters from a-z or A-Z or 0-9 or _ or -.
  • Group Id: Mandatory - is group id of the group into which the data is import.
  • User Type: Mandatory. Please note that Primary Group Administrator account cannot be modfied
  • First Name, Email: Mandatory
  • Last Name: Optional
  • State: 0 => deactivated, 1 = active.
  • OU: OU name as available in OU tree. Mandatory for create and update; ignored otherwise.
  • Organization specific metadata: Optional. One or more columns like "location" and "grade" may appear. This is set of custom user metadata configured for your group.
  • Supervisor: Optional. The "User id" of the supervisor. The supervisor account should be created already (or available before the row referencing the supervisor.
  • Joining date: Optional. The joining date of the user in format date:yyyy-mm-dd.
  • Created / Updated UTC Time: Created / Updated timestamps in UTC - readonly information when exporting. These are ignored during import.
