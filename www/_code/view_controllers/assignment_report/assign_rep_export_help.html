Depending on the options chosen during export, this zip file may contains 3 kind of learning records. Each kind of learning record is exported into one or more .csv file(s) depending on the number of entries.

report-overview-[n].csv

Each row of these csv files is a learning record of one user learning one module. The following fields are exported in these csv files:

Name Description
user loginid This is the unique login id of the user. Either the login id or the email id could be used as a key to uniquely lookup the user from your MIS database.
user name The name of the user.
module name The name of the learning or quiz module.
status The status could be
  • completed: The user has successfully completed the module and scored above the defined pass criteria. If no criteria is defined in the module, by the default pass criteria is 70%.
  • failed: The user has submitted but scored below the pass criteria.
  • pending: The user has not yet submitted.
% Percentage score of the user (score / maxScore x 100).
score The actual score of the user.
maxScore The miximum possible score for this module.
passScore The minimum score required to pass.
time spent The total time spent on this module.
created Date and time of assignment creation.
started Date and time when the user starts the assignment.
ended Date and time when the user completes the assignment.
updated Last updated date and time of the assignment. When a user has not started, updated time will be same as created. After the user completes, the updated time will be same as ended time. If the assignment administrator later updates the assignment after completion by the user, this timestamp could be later than ended time stamp.
email email id of the user to whom the assignment has been assigned.
org class/organization unit the user belongs to in your org structure.
assign type One of the below values:
  • module assignment:This learning record is based on a module assigned to the user.
  • course assignment:This learning record is based on a module inside a course assigned to the user.
  • self assignment:This learning record is based on user self learning.
recordid Unique id of the this learning record. All the id parameters mentioned are useful only in case of building a database of the exported records.
userid Unique id of the user.
assignid Unique id of the assignment. Valid only incase of module assignment.
moduleid Unique id of the module.
course Name of the course. All the course related parameters are valid only if assigntype is "course assignment".
courseid Unique course id
courseassignid Unique course assignment id
courserepid Unique report id of each user doing the course

page-scores-[n].csv

This csv file displays the page level scores of the users who have submitted the module. The following fields are exported:

Name Description
# Serial number.
user loginid See above.
user name See above.
module name See above.
page no Page number of the module for which the score is to be displayed.
page title This field includes information provided in the title section of the page.
score Page score attained by the user.
maxScore Maximum possible page score.
other parameters See above.

feedback-[n].csv

This csv file displays the feedback provided by the user in the questionnaire pages of the module. The following fields are exported:

Name Description
# Serial number.
user loginid See above.
user name See above.
module name See above.
page no See above.
page title See above.
question The question for which the feedback is to be provided.
response Users response to the question.
other parameters See above.
