Depending on the options chosen during export, this zip file may contains 3 kind of learning records. Each kind of learning record is exported into one or more .csv file(s) depending on the number of entries.

Common fields

Each row of these csv files contains the below common fileds:

Name Description
User Id This is the unique user id of the user. Either the User Id or the Email Id could be used as a key to uniquely lookup the user from your MIS database.
User Name The full name of the user.
Email Id Email Id of the user to whom the assignment has been assigned.
Org Organization unit the user belongs to in your org structure.
Meta data fields User metadata configured for your organizations (e.g. location, grade, ...).
Course Name Name of the course. All the course related parameters are valid only if Assign Type is "course assignment".
Module Name The name of the learning or quiz module.
Assign Type One of the below values:
  • module assignment:This learning record is based on a module assigned to the learner.
  • course assignment:This learning record is based on a module inside a course assigned to the learner.
  • module self assignment:This learning record is based on self learning of a module.
  • training:This learning record is based on a external training assigned to the learner.

report-overview-[n].csv

Each row of these csv files is a learning record of one user learning one module. The following fields are exported in these csv files additional to the common fields:

Name Description
Assigned On Date and time of assignment creation.
Started On Date and time when the learner starts the assignment.
Ended On Date and time when the learner completes the assignment.
Last Updated On Last updated date and time of the assignment. When a learner has not started, updated time will be same as created. After the learner completes, the updated time will be same as ended time. If the assignment administrator later updates the assignment after completion by the learner, this timestamp could be later than ended time stamp.
Status The status could be
  • completed: The learner has successfully completed the module and scored above the defined pass criteria. If no criteria is defined in the module, by the default pass criteria is 70%.
  • failed: The learner has submitted but scored below the pass criteria.
  • pending: The learner has not yet submitted.
Achieved % Percentage score of the learner (score / maxScore x 100).
Maximum Score The miximum possible score for this module.
Achived Score The actual score of the learner.
Pass % The minimum score required to pass.
Time Spent (minutes) The total time spent on this module.

page-scores-[n].csv

This csv file displays the page level scores of the learners who have submitted the module. The following fields are exported:

Name Description
Page No Page number of the module for which the score is to be displayed.
Page Title This field includes information provided in the title section of the page.
Maximum Score Maximum possible page score.
Achieved Score Page score attained by the learner.

feedback-[n].csv

This csv file displays the feedback provided by the learner in the questionnaire pages of the module. The following fields are exported:

Name Description
Page No Page number of the module for which the score is to be displayed.
Page Title This field includes information provided in the title section of the page.
Question The question for which the feedback is to be provided.
Response Users response to the question.
