Define the course properties.

  • name: Mandatory - enter a name for your course.
  • image: Mandatory - enter a URL for the course icon that will be displayed when this course is searched.
  • description: Provide a short description which will help others in the group to understand the purpose of this course.
  • content: Mandatory - course content in JSON format(look following format for reference).

Course content JSON format

Course is an array of course modules. Each course module describes a lesson, a quiz or a module/folder which futher containes lessons, quizes and modules.

Hint: http://www.jsoneditoronline.org/ provides a good tool for editing JSON strings.

Attribures of course module

Ever course module must contain the attributes "id", "name" and "type". "id" unniquely indentifies the course-module. All these attributes are strings. Example:

[
  {"id": "1", "name": "course module sample 1", "type": "module"},
  {"id": "2", "name": "course module sample 2", "type": "module"}
]

Course module types

Each course module can be of type "module" or a "lesson".

  • "lesson" is a leaf node in the tree. Clicking on this will open the lesson. When defining a couse-module of "type": "lesson", the attribute "refid" needs to be provided. This is the lessonid if the lesson to be launched when clicking on the link.
  • "module" is just a container/folder within the course to organize set of other modules and lessons.
[
  {"id": "lesson1", "name": "Lesson 1", "type": "lesson", "refid": 1231},
  {"id": "folder1", "name": "Folder containing some lessons", "type": "module"},
  {"id": "folder1.lesson1", "name": "Lesson 2", "type": "lesson", "refid": 1232},
  {"id": "folder1.lesson2", "name": "Lesson 3", "type": "lesson", "refid": 1233},
  {"id": "folder1.lesson3", "name": "Lesson 4", "type": "lesson", "refid": 1234}
]

Defining a course tree (parent/child relationship)

"parentid.childid": id attribute is used to determine parent child relationship. id of the child element should always begin with the id of parent element and a ".".

Assume a module with "id": "1" has three lessons/quizes defined under it. Then the children's id is derived from the parent. First, second and third child id should begin with "1.". For example "id":"1.1", "id":"1.2".

[
    {"id": "1", "name": "module name", "type":"module", "icon":"module"},
    {"id":"1.1", "name": "lesson/quiz/module", "type":"lesson/quiz/module", "icon": "lesson/quiz/module", "refid": "1234"},
    {"id":"1.2", "name": "lesson/quiz/module", "type":"lesson/quiz/module", "icon":"lesson/quiz/module", "refid": 1234 },
    {"id":"1.2.1", "name": "lesson/quiz/module", "type":"lesson/quiz/module", "icon":"lesson/quiz/module", "refid": 12345 },
    {"id":"1.2.2", "name": "lesson/quiz/module", "type":"lesson/quiz/module", "icon":"lesson/quiz/module", "refid": 12346 },
    {"id":"1.3", "name": "lesson/quiz/module", "type":"lesson/quiz/module", "icon":"lesson/quiz/module", "refid": 1235 }
]

Defining icons for the course modules

"icon" field is not mandatory to be specified. By default, the icon is chosen depending on the "type".

Course icon can be customised by providing an image url. Three predefined icons are provided: "module", "lesson" and "quiz" which may be used instead of specifying full icon url.

[
    {"id": "1", "name": "xx", "type":"module", "icon":"https://xyz.png"},
    {"id": "1", "name": "xx", "type":"lesson", "icon":"quiz"}
]
