Module Properties

This is automatically shown to you when a new module is created. You can always press the property toolbar icon to get to this window again.

  • name: Mandatory - enter a name for your module.
  • Grade, Subject: Choose the grade and subject for which this module is created.
  • image: Choose the icon that will be displayed when this module is searched. Try 'Custom' when you are not satisfied with the provided options.
  • custom url: If you choose 'Custom' in image field, you can enter URL (link) of the custom image. The URL could be an image uploaded within the system or a link to external source in internet.
  • PDF URL: Mandatory - enter the URL of PDF file to create the module.
  • Single Page: If checked, the complete PDF content is generated in one page. Otherwise, each PDF page will be rendered in one Nittio Page
  • description: Write a description which will help others in the group to understand the purpose of this module.
  • keywords: Some keywords to help in search. You could use this categorize your content.
  • estimated time (minutes): Estimated time in minutes (between 1 and 600) to complete this learning module. When this learning module is sent as an assignment, the assignment duration is per-filled with the estimated time if available.
  • pass score %: Minimum pass score in percentage (between 0 and 100 - both inclusive) for passing this module. Set this to zero to consider any score as passed. Clean the value to go back to the default value.
  • score limit: This parameter is specific to question bank. When the question bank is distributed, a random subset of questions will be chosen from the bank. The total score of the chosen questions will be kept equal to the score limit. If the maximum scores are different for different pages, it is possible that the maximum score of chosen pages is slightly less than the score limit.
  • discussion topic: provide name of the discussion topic which should be displayed when the learner clicks on discussion forum icon from this lesson. This could be further change at page level. This field is the default at lesson level.
  • Template specific attributes:

    These properties customizes the module editor and are expected to be configured in a template module.

    • styles: define custom CSS classes to be used in styling the module content.
    • background images: Provide a list of background images (to be displayed in change look dialog) as a JSON array of strings.
    • module icons: Provide a list of module icons (to be displayed in module properties) as a JSON array of strings.
    • page types: Provide page types and layout as a JSON object.
    • animation schemes: Provide animation schemes as a JSON object.
